# Decision Log: LeetcodeCLI Product Brief

Created: 2026-06-11

## Decisions

- Create a product brief for a LeetCode-focused CLI tool.
- Treat the first stated implementation direction as Go with the Cobra framework.
- Capture the initial value proposition as terminal access to LeetCode profile and solved-question statistics.
- Position the tool as a public developer tool that anyone can use, not only a personal utility.
- Require login before viewing stats.
- Treat the initial command shape as `leetcode stats username`.
- Use pretty terminal tables for v1 output.
- Keep v1 as a simple stats reviewer, not a practice companion.
- Treat the audience as broad across LeetCode users, while optimizing v1 for terminal-native individual developers.
- Make `leetcode stats` default to the logged-in user.
- Use `leetcode stats username` for explicit username lookup.
- Use browser-based login as the v1 authentication approach.
- Support Windows, macOS, and Linux in v1.
- Browser login is the only supported authentication method; v1 will not support manual session-cookie paste/import fallback.
- Support viewing any public LeetCode profile with `leetcode stats username`.

## Source Notes

- User described the product as: a LeetCode CLI through which a user can see their profile, number of questions solved, number solved in each language, and related stats.
